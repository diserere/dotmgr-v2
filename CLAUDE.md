# dotmgr — контекст для AI-ассистента

## О проекте

dotmgr — CLI-инструмент для синхронизации конфигов между Linux-машинами. Bash bootstrap + Python ядро.

## Текущая фаза

**Pre-development.** Фиксация требований (PRD), архитектуры и процессов. Код ещё не пишется.

## Ключевые правила

1. **No direct commits to `main`.** Все изменения — в feature-ветках. Исключение — самые первые коммиты инициализации репо.
2. **Conventional Commits.** Формат: `type(scope): описание`. Обязателен `Co-Authored-By` для коммитов агента.
3. **PR с локальным merge.** Открываем PR на GitHub для истории обсуждения. Merge — локально в main.
4. **Язык:** документация и коммиты — на русском. Технические термины — на английском.
5. **Context-as-Code.** Документация (`/docs/`) — главный источник истины о проекте.

## Архитектура (предварительно)

- **2 репозитория:** публичный (код инструмента) + приватный (конфиги пользователя)
- **Stow-подобная структура** хранения конфигов в git-репо
- **Bootstrap:** `wget ... | bash` → установка зависимостей + dotmgr
- **MVP:** `fetch` + `apply` (pull-only). Push — в V2.

## Источники контекста

- `docs/PRD.md` — требования и MVP scope
- `docs/AGENT_WORKFLOW.md` — процесс разработки
- `docs/CONTEXT_AS_CODE.md` — принципы работы с контекстом
- `docs/CONTRIBUTING.md` — git workflow, коммиты, языковая политика

## Команды MVP (предварительно)

| Команда | Описание |
|---|---|
| `dotmgr init` | Bootstrap: установка тула + создание/клонирование config-репо |
| `dotmgr fetch` | `git pull` конфигов из remote |
| `dotmgr apply [group...]` | Применение конфигов на ФС |
| `dotmgr list` | Список доступных групп |
