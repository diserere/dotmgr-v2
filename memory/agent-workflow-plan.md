# Agent Workflow: гибридный подход для фазы Architecture

## Контекст

Принято решение перейти к фазе Architecture & Design (см. ROADMAP). Обсуждался формат работы:

1. Многоагентный подход (раздельные агенты с ролями BA / SA / Developer / QA)
2. Моноагентный (текущая сессия, накоплен контекст)

## Решение

Выбран **гибрид**:

- **Оркестратор (BA)** — текущий агент, владеет контекстом проекта.
- **Субагент (SA)** — Task-агент, получает промпт от оркестратора, пишет черновик ARCHITECTURE.md на основе /docs/.
- **Человек + оркестратор** — ревью результата SA, правки, фиксация.

## Обоснование

| За | Против |
|---|---|
| SA не тащит шум предыдущих обсуждений | SA не знает нюансов, не зафиксированных в docs/ |
| Чистый взгляд может выявить пробелы в docs/ | Требует качественного промпта от оркестратора |
| Проверяем передачу контекста через docs/ (Context-as-Code) | — |
| Сохраняем интерактивность на этапе ревью | — |

## Выводы для следующих фаз

Подход применим для документации и архитектуры. Для реализации (код) может потребоваться другое разделение — например, полное разделение ролей Dev + QA.

## Ретро: завершение фазы Architecture (гибрид)

### Что получилось

- **Передача контекста сработала** — субагент-SA, получив промпт с отсылкой к `/docs/`, сгенерировал полноценный 772-строчный черновик, который лёг в основу итогового документа.
- **Ревью-цикл эффективен** — человек нашёл ~14 замечаний (оркестратор — ещё ~4), правки внесены в 2 раунда. Ни одно замечание не потребовало полного переписывания.
- **Context-as-Code подтверждён** — SA не знал истории обсуждений PRD, но это не помешало: всё нужное было в PRD + ROADMAP. Пробелы, которые SA подсветил (TODO), стали точками для уточнения.

### Что могло быть лучше

- **Промпт субагенту можно было детальнее** — SA не учёл некоторые наши ранние решения (например, venv внутри tool/), хотя они были в CLAUDE.md. В следующий раз стоит явно перечислять «неочевидные решения» в промпте.
- **Даунтайм на ожидание SA** — генерация 772 строк заняла время. Для мелких задач (фикс бага) проще сделать самому, не запуская субагента.

### Вывод

Гибридный подход (оркестратор → субагент → ревью) оправдал себя для фазы Architecture. Для следующей фазы (Implementation MVP, задача 1: install.sh) планируется повторный запуск субагента-Developer.
