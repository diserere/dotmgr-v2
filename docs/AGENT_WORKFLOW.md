# Агентский рабочий процесс

Этот документ определяет политику взаимодействия AI-агентов с человеком в рамках данного репозитория.

## Человек в контуре (Human-in-the-loop)

Агент автономно выполняет **read-only** операции (поиск, чтение, запуск безопасных команд) без одобрения. Любая **мутирующая** операция (запись файлов, запуск команд, меняющих состояние) требует явного одобрения человеком через инструмент выполнения.

## Ограничения агентов

*   **Изоляция:** Агент не должен изменять хост-систему без одобрения.
*   **Безопасность:** Любые действия, изменяющие состояние проекта, требуют явного подтверждения.
*   **Область действия:** Агент работает в дереве проекта, если не указано иное.

## Процесс разработки и ревью

1.  **Автор-агент** открывает PR на GitHub для фиксации истории изменений и обсуждений.
2.  **Ревьюер-агент** (или человек) проводит ревью, используя комментарии в PR как артефакты коммуникации.
3.  **Автор-агент** применяет исправления в той же ветке.
4.  Цикл повторяется до решения всех замечаний.
5.  **Локальное слияние:** После одобрения PR, слияние выполняется **локально** в ветку `main` без использования squash-merge, чтобы сохранить историю. PR закрывается автоматически.

## Роли

*   **Автор-агент** — открывает PR.
*   **Ревьюер-агент** — делает ревью.
*   **Человек** — финальное одобрение (MITM).

## Definition of Done (Критерии готовности)

PR готов к ревью, если:
*   Новая функциональность покрыта тестами.
*   Обновлена документация (согласно концепции Context-as-Code).
*   Сообщения коммитов соответствуют Conventional Commits.
*   Документация на русском (согласно языковой политике).
*   CI проходит успешно.
